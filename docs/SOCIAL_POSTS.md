## X

我做了一个本地小工具：BLCaptain Codex Probe CLI。

它不是替代官方 dashboard，也不碰登录态。

你把 Codex `/status` 贴给 Codex，让它调用本地 CLI，就能得到一份报告：

- 为什么这次任务贵
- 怎么降配
- 什么时候该停
- 当前上下文、5 小时额度、7 天额度分别是否健康
- Skill / 输出有没有 AI 味、插件风险和敏感信息

这版补了脱敏 `/status` 样本库，用测试覆盖继续、降配、停止三类决策。

最适合长会话、重度 Codex 用户、内容创作者和轻开发交付。

开源：  
https://github.com/dososo/BLCaptain-Codex-Probe-CLI

## 小红书

标题备选：

- 我做了个 Codex 用量体检工具：告诉你什么时候该停
- Codex 越聊越贵？我做了个本地分析 CLI
- 把 `/status` 贴给 Codex，让它告诉你为什么贵

正文：

最近我自己用 Codex 做项目时，最明显的感觉不是「不会提示词」，而是会话越来越长以后，很难判断：

- 这次到底为什么贵？
- 是不是该换小模型？
- 该继续做，还是先停下来开新会话？
- 5 小时额度和 7 天额度看起来还健康，但当前上下文是不是已经太重？
- Skill / 输出是不是 AI 味太重，或者带了不该有的插件风险？

所以我做了一个本地工具：**BLCaptain Codex Probe CLI**。

现在它不只是生成一份普通报告，而是多了一个 `doctor` 一键体检：

你把 Codex `/status` 贴给 Codex，让它调用本地 CLI，就能生成三份 Markdown 报告：

- doctor 汇总报告
- 任务级用量报告
- Skill / 输出质量体检报告

报告最前面会给一张「决策卡片」，直接回答四件事：

- 建议继续、降配，还是停止
- 为什么这次贵
- 下一步怎么降配
- 什么时候该停下来开新会话

它现在也会单独解析 Codex 桌面版 `/status` 里的几个关键信息：当前上下文剩余、已使用 token / 上限 token、5 小时额度、7 天额度和重置时间。

这版我还补了一个脱敏 `/status` 样本库，用来覆盖几种常见场景：

- 额度和上下文都健康，可以继续
- 当前上下文偏重，建议降配或开新会话
- 5 小时额度偏低，只做收尾验证
- 7 天额度偏低，建议停止非必要任务
- 英文格式 `/status`，也能解析出关键字段

这些样本会进入自动化测试，避免工具只是在 README 里说得好听。

Skill / 输出体检这边，会检查 AI 味、插件风险、敏感信息、缺失验收标准和隐私边界缺口。

边界我也写得很死：

- 不登录 OpenAI
- 不读浏览器 cookie
- 不碰 token 和系统凭据
- 不上传云端
- 不承诺省钱或无限额度

它更像一个「Codex 使用刹车片」：不是帮你绕过限制，而是提醒你别在长会话里无意识烧上下文。

适合重度 Codex 用户、内容创作者、轻开发交付和经常写 Skill / README / 代码的人。
