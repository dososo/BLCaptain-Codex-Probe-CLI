# 经验记录

## README 示例截图必须和公开样本口径一致

- 公开截图中的字段值会被用户当成产品事实；示例模型名、时间、置信度等不能混用旧版本或内部占位命名。
- 生成 README 截图前，先检查样本、示例报告、Dashboard HTML 和 PNG 是否来自同一批数据。
- 模型字段必须说明来源：CLI 原样展示数据源提供的 `model` 字段，不做猜测或强制改写。

## README 的 Codex 桌面版入口不能只写一句话

- 普通用户需要按场景复制：安全 demo、分析本地历史、只分析一次 `/status`。
- 每段提示词都要写清楚报告路径、是否读取真实历史、隐私边界和最后要 Codex 解释什么。
- 顶部最快入口和 `docs/CODEX_DESKTOP_PROMPT.md` 要保持一致，避免用户从不同位置看到不同用法。
- 不要用“开发者安装”把普通用户挡在外面；README 应该优先提供“让 Codex 帮我安装这个仓库”的自然语言入口，再提供终端命令。
