# BLCaptain Codex Probe 使用辅助 Skill

## 何时使用

当用户明确提供 Codex `/status` 文本、Skill 文件、提示词或 AI 输出，并要求进行本地用量分析或质量体检时使用。

## 不适用场景

- 用户要求读取浏览器、登录态、系统凭据、钥匙串或私密目录。
- 用户要求修改 Codex 请求链路、订阅、地区限制或官方计费规则。
- 用户要求安装、开启或推荐未经确认的外部插件。
- 用户要求承诺省钱、提升额度或保证平台通过。

## 输入要求

只处理用户在当前任务中明确提供的文本或本地文件路径。

如果用户粘贴 `/status`，先保存到 `.probe/` 目录。保存前需要脱敏：

- API key
- cookie
- token
- 邮箱
- 手机号
- 会话标识

## 工作流程

1. 确认当前目录是 `BLCaptain-Codex-Probe-CLI` 项目目录。
2. 检查本地虚拟环境和 `codex-probe` 命令是否可用。
3. 导入用户提供的 `/status` 或手工 JSON。
4. 生成任务级用量报告。
5. 对用户提供的 Skill、提示词或输出文本运行质量体检。
6. 用普通话总结报告路径、主要风险、降配建议和停止线。

## 输出要求

最终回复只包含：

- 报告路径
- 主要风险
- 建议继续、降配或停止
- 已执行的验证结果

不要复述完整敏感输入。不要输出原始密钥、cookie、token、邮箱或手机号。

## 验收标准

- 用量报告写入 `reports/usage-report.md` 或用户指定路径。
- 质量体检报告写入 `reports/skill-lint-report.md` 或用户指定路径。
- 报告包含风险标签、严重度、证据片段、建议和证据来源。
- 脱敏后的报告不包含原始敏感信息。
- 如执行删除命令，再次报告应返回 `NO_USAGE_DATA`。

## 隐私与安全

- 不上传云端。
- 不读取浏览器、登录态、cookie、token、钥匙串或系统凭据。
- 不修改用户未明确提供的文件。
- 不安装或修改插件。
- 不承诺节省费用、提升额度或替代官方用量系统。
- 用户要求删除时，只清理本工具产生的本地业务数据，并保留必要审计记录。
